# Day 32 – Route Params, Query Params & REST APIs

## What I Learned
- Difference between route params and query params
- How to access req.params and req.query
- How to structure REST APIs properly

## Key Concepts
- Route params identify specific resources
- Query params are used for filtering
- REST APIs follow consistent patterns

## What I Implemented
- Dynamic routes using :id
- Query parameter handling
- Basic REST-style users API

## REST Structure Example
GET /users
GET /users/:id
POST /users
PUT /users/:id
DELETE /users/:id

## Why This Matters
- Standard backend design
- Important for interviews
- Required for full MERN integration
