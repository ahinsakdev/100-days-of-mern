# Day 22 – Controlled Components & Forms in React

## What I Learned
- Why React uses controlled components
- How inputs are linked to state
- How forms are handled in React

## Key Concepts
- Controlled input gets its value from state
- onChange updates state on every keystroke
- React becomes the single source of truth

## Input Handling Flow
- State holds input value
- Input displays state value
- User types → onChange fires
- State updates → UI re-renders

## Form Submission
- Default browser submit is prevented
- React handles submit logic manually

## Notes
- Avoid uncontrolled inputs
- Always bind value and onChange
